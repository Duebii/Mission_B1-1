# 🥬 야채도감 — 서비스 기획 대화록

---

## 1. 기획 배경

> 야채를 즐기는 편인데, 좋아하는 대로 먹다 보니 어떤 성분이 있는지 궁금해졌다.  
> 찾아보니 정보가 너무 흩어져 있고, 유효성분 이름도 낯설어서 정신없었다.  
> 이것이 **야채도감** 서비스를 기획하게 된 배경이다.

---

## 2. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 야채도감 |
| 형태 | 모바일 앱 |
| 핵심 문제 | 정보의 파편화 + 전문 용어로 인한 낮은 접근성 |
| 목표 | 야채를 즐기는 사용자가 쉽고 재미있게 건강 정보를 탐색할 수 있는 앱 |

---

## 3. 타겟 사용자

- **4050 주부** — 건강 및 요리에 관심이 많은 층
- **2030 여성 1인 가구** — 직관적 비주얼 정보를 선호하는 층

---

## 4. 차별화 포인트

1. **이모지 및 픽토그램 중심 정보 제공** — 텍스트 의존도를 낮추고 직관적으로
2. **동일 유효성분 기반 야채 연결** — "이 성분이 들어있는 다른 야채" 탐색 유도
3. **자연스러운 정보 탐색 유도** — 강요 없이 흐르듯 연결되는 UX

---

## 5. 필수 포함 항목

> 복잡한 영양학 정보를 쉽고 직관적으로 이해할 수 있도록 사용자 배려

---

## 6. 핵심 화면 구성 (1차 목업)

### 화면 1 — 홈 · 탐색
- 상단: 서비스명 + 검색창
- 효능 태그 필터 (면역력 / 피부 / 두뇌 / 뼈 / 심장 등)
- 제철 야채 카드 그리드 (이모지 + 이름 + 효능 뱃지)
- 하단 탭바: 홈 / 탐색 / 성분 / 저장

### 화면 2 — 야채 상세
- 야채 이모지 + 이름 + 효능 태그
- 주요 유효성분 목록 (아이콘 + 한 줄 설명 + 함량 바)
- "같은 성분이 있는 야채" 추천 (가로 스크롤 카드)

### 화면 3 — 성분 상세
- 성분명 (한글) + 영문명 + 분류
- 이런 효과가 있어요 (픽토그램 + 쉬운 설명)
- 이 성분이 들어있는 야채 목록 (필 형태)

---

## 7. 디자인 방향 논의

**1차 목업 피드백:** "너무 식상하다"

**논의 중인 방향:**
- 어떤 느낌을 원하는가? (감성적·따뜻한 / 깔끔·쿨한 / 위키·백과 / 엉뚱·재미있는)
- "도감"하면 떠오르는 이미지는? (어릴 때 본 식물 도감 책 / 포켓몬 도감 / 자연 다큐 감성 / 인스타 감성 콘텐츠)

> ⚠️ **미결 사항** — 디자인 방향 및 비주얼 톤 최종 결정 필요
